# .ai 项目辅助系统

这个隐藏文件夹（`.ai`）用于记录项目的各种信息、问题、解决方案和重要提示。

## 文件说明

### 📋 PROJECT_OVERVIEW.md
项目的基本信息总结：
- 项目描述和核心特性
- 项目结构和目录说明
- 依赖关系
- 数据库支持
- 主题生态

**何时使用**: 需要快速了解项目整体结构时

### 🔧 ISSUES_AND_SOLUTIONS.md
所有遇到的问题和解决方案的详细记录：
- 问题描述
- 根本原因分析
- 采用的解决方案
- 关键提交/修改
- 经验教训

**何时使用**: 
- 遇到类似问题时查看历史解决方案
- 理解为什么要这样做
- 学习项目的演进过程

### ⚠️ IMPORTANT_NOTES.md
关键代码位置、常用命令、常见错误和解决方案：
- 关键文件位置和内容
- 常用命令速查
- 文件修改检查清单
- 常见错误和解决
- 快速参考表

**何时使用**:
- 修改代码前做检查
- 查询常用命令
- 遇到错误时查阅

## 使用规则

### 对于AI助手（我自己）

**每次开始新对话时**:
1. 首先读取 `.ai/PROJECT_OVERVIEW.md` 了解项目
2. 读取 `.ai/ISSUES_AND_SOLUTIONS.md` 了解历史问题
3. 读取 `.ai/IMPORTANT_NOTES.md` 了解重要事项

**每次修改代码前**:
1. 检查 `.ai/IMPORTANT_NOTES.md` 中的"文件监控清单"
2. 参考相关的历史问题和解决方案
3. 按照"文件修改检查清单"操作

**每次完成任务后**:
1. 更新 `.ai/ISSUES_AND_SOLUTIONS.md`，记录：
   - 遇到的新问题
   - 采用的解决方案
   - 失败的尝试（标记为❌）
   - 成功的方案（标记为✅）
   - 学到的经验教训
2. 更新 `.ai/IMPORTANT_NOTES.md`，如有新的：
   - 关键代码位置
   - 常见错误
   - 快速参考

### 对于用户

- ✅ 可以查看 `.ai` 文件夹了解项目的各种信息
- ✅ 可以从 `.ai/ISSUES_AND_SOLUTIONS.md` 了解项目的问题历史
- ✅ 可以从 `.ai/IMPORTANT_NOTES.md` 学习最佳实践
- ❌ 请勿修改这些文件（由AI助手自动维护）
- ❌ 请勿删除这些文件（它们是项目的知识库）

## 文档维护规范

### 更新时机

| 事件 | 更新哪个文件 |
|------|----------|
| 发现新问题 | ISSUES_AND_SOLUTIONS.md |
| 采用新解决方案 | ISSUES_AND_SOLUTIONS.md |
| 遇到新错误 | IMPORTANT_NOTES.md |
| 学到新命令 | IMPORTANT_NOTES.md |
| 项目结构变化 | PROJECT_OVERVIEW.md |

### 标记规范

在记录时使用以下标记：

- ✅ 成功的解决方案
- ❌ 失败的尝试或已撤销的方案
- ⚠️ 需要注意的地方
- 📋 信息记录
- 🔧 解决方案
- 📝 笔记或说明

### 版本控制

- 不要频繁commit .ai文件夹的更改
- 可以在git ignore中添加 `.ai/` 来避免频繁提交（可选）
- 或者每个会话结束后统一提交一次
